Efficient Information Gathering for Penetration Testing

Information gathering is a crucial step in penetration testing. Without properly gathering information, you risk missing important parameters required for vulnerability testing. In this blog, I will outline essential tools for information gathering and provide a script to streamline this process.

Credit to Lostsec

Target Setup

  1. Create a directory for your target: 
    mkdir google
  2. Prepare a text file with domains in scope: 
    echo "example.com" > domains.txt

Finding Subdomains

Use subfinder to discover subdomains:

subfinder -dL domains.txt -all -recursive -o subdomains.txt

Check the total number of subdomains:

cat subdomains.txt | wc -l

Important Rule: Never rely on a single tool for results. To dig deeper, use crt.sh:

curl -s https://crt.sh/\?q\=google.com\&output\=json | jq -r '.[].name_value' | grep -Po '(\w+\.\w+\.\w+)$' | tee -a subdomains.txt

This command queries crt.sh for TLS/SSL certificates and extracts subdomains, providing comprehensive results.

Checking Alive Subdomains

Identify active subdomains:

cat subdomains.txt | httpx-toolkit -l subdomains.txt -ports 443,80,8080,8000,8888 -threads 200 > subdomains_alive.txt

Port Scanning

Scan ports using naabu:

naabu -list subdomains.txt -c 50 -nmap-cli 'nmap -sV -sC' -o naabu-full.txt

Directory Search

Search directories on alive subdomains:

dirsearch -l subdomains_alive.txt -x 500,502,429,404,400 -R 5 --random-agent -t 100 -F -o directory.txt -u /home/kali/oneListForall/onelistforallshort.txt

Note: This process might take an hour or more depending on the target size.

Filtering Parameters

Check parameters:

cat params.txt | wc -l
cat params.txt | uro -o filterparam.txt
cat filterparam.txt | grep ".js$" > jsfiles.txt
cat jsfiles.txt | uro | tee -a jsfiles.txt

Secret Finder

Identify sensitive data such as API keys and tokens:

  1. Clone SecretFinder: 
    git clone https://github.com/m4ll0k/SecretFinder.git
  2. Run the script: 
    cat jsfiles.txt | while read url; do python3 /home/kali/SecretFinder/SecretFinder.py -i $url -o cli >> secret.txt; done
  3. Install missing modules if required: 
    sudo pip3 install jsbeautifier
  4. Grep for secrets: 
    cat secrets.txt | grep aws

Sorting Parameters

Sort and filter parameters:

# sort.py
with open('filterparam.txt', 'r') as file:
    lines = file.readlines()

lines.sort()
lines = lines[:100000]

with open('sorted_param_100000.txt', 'w') as file:
    file.writelines(lines)

Run the script:

python3 sort.py

Nuclei Scanning

Use nuclei with sorted parameters:

nuclei -list sorted_param_100000.txt -c 70 rl 200 -fhr -lfa -t /home/kali/Custom-Nuclei-Templates -o nucleigoogle.txt -es info

Clone custom templates for better results:

git clone https://github.com/0xKayala/Custom-Nuclei-Templates.git
